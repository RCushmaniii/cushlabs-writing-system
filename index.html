Writing System

Clone your brand's voice once—unleash AI to dominate every page, email, and pitch.

Load your style, audience, and truth as simple files—then fire off flawless, on-brand content without endless prompting.

What This System Does

This writing system prevents the most common failures in AI-generated content:

  • Generic voice — Every business sounds the same
  • Invented claims — AI hallucinates offers, pricing, or proof points
  • Inconsistent messaging — Different assets contradict each other
  • Format drift — Structure degrades over time

It does this by separating what to say (context) from how to say it (skills), and enforcing both through structured prompts and validation.

Core Components

Context Profiles

Structured JSON files that define voice DNA, ideal client profile, business offers, and claims constraints. These are the single source of truth for all content.

Files: voice-dna.json, icp.json, business-profile.json, claims-policy.json

Creator Prompts

Structured interview prompts that extract context from the business owner. These define how truth enters the system.

Output: Context JSON files

Content Skills

Format-specific expertise packages for landing pages, emails, case studies, social posts, and more. Skills define structure and strategy—not voice or claims.

Input: Context + topic → Output: Formatted content

Knowledge Base

Reference content, templates, and examples organized by trust level. Gold-standard content is safe to imitate; drafts require verification.

Categories: Gold-standard, templates, drafts, notes

How It Works

  1. Extract context — Run creator prompts to build context profiles
  2. Select skill — Choose the content type you need (landing page, email, etc.)
  3. Provide topic — Specify what you want to write about
  4. Generate content — Skill loads context and produces structured output
  5. Validate quality — Preflight checks ensure accuracy and consistency

Context is authored once and reused across all content types. Skills ensure format consistency without reinventing voice or claims.

Key Principles

  • Context over prompts — Truth lives in structured files, not ad-hoc instructions
  • Separation of concerns — Voice, audience, offers, and format are independent
  • Single source of truth — Context files are canonical; everything else derives from them
  • Explicit constraints — What not to say is as important as what to say
  • Validation over trust — Preflight checks catch errors before publishing
